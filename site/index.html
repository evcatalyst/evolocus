Run locally

Start the demo evaluator with no network and no real corpus.

Demo mode uses unmistakably synthetic records and supports the full review workflow: queue selection, blinded labels, save, skip, flag, corrections, metrics, and exports.

Demo mode

python -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
EVOLOCUS_MODE=demo streamlit run dashboards/app.py

Local LOCUS Parquet mode

EVOLOCUS_MODE=local \
EVOLOCUS_DATA_GLOB='data/raw/locus-v1/<revision>/**/*.parquet' \
streamlit run dashboards/app.py

Architecture

Polars reads the corpus; SQLite preserves human review state.

The evaluator keeps raw LOCUS fields intact, adds derived metadata separately, samples bounded queues, and writes reviewer actions as append-only events. DuckDB, embeddings, Census enrichment, RAG, and maps remain optional future work after the evaluation layer is proven.

  1. 1 LOCUS Parquet or synthetic demo records.
  2. 2 Polars lazy corpus service with bounded retrieval.
  3. 3 Reproducible queue sampler with pinned seed and revision.
  4. 4 Streamlit blinded human review workflow.
  5. 5 Append-only SQLite review events and content-free exports.
